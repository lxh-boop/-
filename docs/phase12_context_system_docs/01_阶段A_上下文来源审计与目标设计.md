# Phase 12-A：上下文来源审计与目标设计

## 本阶段目标

本阶段先不要大规模改代码，重点是把当前项目中的上下文来源查清楚，并形成 ContextManager 的落地设计。

目标：

```text
找出上下文从哪里来
找出上下文现在怎么传
找出哪些上下文进入了 LLM
找出哪些上下文被工具使用
找出哪些上下文来自 session_state
找出 Artifact / Approval / ToolResult 如何传递
生成上下文来源审计报告
建立 ContextManager 设计草案
```

---

## 一、允许做

1. 搜索和阅读代码；
2. 新增审计报告；
3. 新增少量只读检查脚本；
4. 设计 ContextBundle 字段；
5. 设计 ContextPolicy；
6. 设计接入点；
7. 可以新增 `agent/context/README.md` 或设计文件；
8. 可以新增空包或类型草案，但不要接入主链。

---

## 二、禁止做

1. 不接入 executor 主链；
2. 不改 ToolExecutor 行为；
3. 不改 Write Gateway；
4. 不改 UI 逻辑；
5. 不改数据库 schema；
6. 不重写工具；
7. 不实现 MemoryManager；
8. 不实现 MessageBus；
9. 不删除旧 context/session_state；
10. 不改变业务结果。
