# Phase 13-A：通信链路审计与消息协议设计

## 本阶段目标

本阶段先不要大规模改代码，重点是把当前项目里的通信链路查清楚，并设计 AgentMessage / MessageBus 的目标协议。

目标：

```text
找出现在有哪些函数间 dict 传参
找出 ToolResult / UnifiedToolResult 如何传递
找出 ContextBundle 如何进入 executor / tool
找出 Approval / pending plan 如何传递
找出 Artifact ref 如何传递
找出 UI 如何读取 agent 结果
找出错误和 warning 如何传递
设计标准 AgentMessage 协议
```

---

## 一、允许做

1. 搜索和阅读代码；
2. 新增通信链路审计报告；
3. 设计 AgentMessage 字段；
4. 设计 MessageType；
5. 设计 MessagePolicy；
6. 设计 MessageBus 接入点；
7. 可以新增 `agent/communication/README.md` 或设计草案；
8. 可以新增空包或类型草案，但不要接入主链。

---

## 二、禁止做

1. 不接入 executor 主链；
2. 不改 ToolExecutor 行为；
3. 不改 Write Gateway；
4. 不改 ContextManager；
5. 不改 UI 逻辑；
6. 不改数据库 schema；
7. 不实现 MemoryManager；
8. 不实现 ReAct；
9. 不实现 Reflection；
10. 不实现 Multi-Agent Handoff；
11. 不删除旧 dict 传参；
12. 不改变业务结果。
